Migrasi ke linux untuk Pemula

Introduction

Pada kesempatan kali ini saya akan mulai menulisakn artikel berseri mengenai pengalaman saya menggunakan linux kurang lebih hampir 10 tahun hingga sekarang, dimana pada saat itu pertama kalinya saya mengnal sistem operasi open source linux dan pertama kalinya juga saya bisa mengoperasikan Komputer itupun karena saya bisa mengenyam pendidikan ke jenjang bangku SMK. Tanpa latar belakang menyukai dunia IT karena memang terbatasnya pembelajaran di bidang IT sewaktu saya duduk di bangku SMP tapi begitu lulus SMP saya memutuskan untuk meneruskan pendidikan ke jenjang SMK dengan mengambil jurusan Teknik Komputer dan Jaringan.

Tanpa mengenal lebih jauh dan menggunakan Windows saya membeli laptop dan menginstall sistem operasi linux Ubuntu.

Masih teringat sistem operasi pertama di Notebook saya sewaktu SMK saya menginstall Linux UBUNTU 9.04 dan mulai mengutak-atik tampilan desktop dan belajar dasar-dasar pengetahuan sistem operasi berbasis open source. Sebagai seorang yang masih awam dan pertama kali tau sistem operasi bernama Ubuntu, ada beberapa point yang saya lakukan untuk pertama kalinya menggunakan Ubuntu, ini berlaku juga untuk kamu yang baru saja migrasi dari sistem operasi lama kamu misalnya dari sistem operasi Windows lalu migrasi dengan menggunakan Linux:

  1. Pilih distro yang cocok untuk kamu yang masih pemula dan pilih jenis distro yang umum.

    Ini saya rasa sangat penting, sebagai seorang pemula cobalah memilih distro linux yang banyak di pakai oleh kebanyakan seorang pemula untuk mengawali perjalanan migrasi dan menyesuaikan dengan sistem operasi yang baru, disini saya contohkan menggunakan distro linux turunan dari Debian misalnya ubuntu , Linux Mint dsb. Mengapa demikian ? jawabannya, karena untuk memudahkan segala sesuatu yang berhubungan dengan pekerjaan lama kamu sewaktu masih menggunakan sistem operasi Windows maka kamu akan dengan mudah mencari alternatif aplikasi ataupun kebutuhan aplikasi yang mendukung pekerjaan kamu saat masih menggunakan Windows dan di sistem operasi turunan Debian akan dengan mudah kita temui aplikasi, tutorial, ataupun panduan untuk pengoperasian sistem operasi tersebut.

  2. Migrasi ke Linux karena memang butuh, bukan karena terinspirasi sesuatu.

    Pastikan kamu yang semula terbiasa dengan sistem operasi semacam Windows karena kamu memang berniat mengganti sistem operasinya dengan alasan mungkin saat kamu menggunakan sistem operasi Windows kamu menggunakan yang bajakan lalu kamu sadar akan software bajakan itu tidak baik, lalu kamu berpindah menggunakan Linux dengan tujuan linux itu open source. Itu bagus dan saya sangat anjurkan untuk migrasi!

    Pastikan kamu juga pakai linux bukan karena terinspirasi setelah melihat tutorial atau film bertema Hacking dimana di tampilkan adegan Hacking di film tersebut dengan menggunakan sistem operasi misalnya Kali Linux lalu dengan inspirasi pertama itu kamu lalu memutuskan untuk belajar teknik-teknik Hacking dan menginstall sistem operasi Kali Linux, tanpa dasar dan pemahaman mengenai sistem operasi open source itu merupakan langkah yang tidak tepat. Kamu harus sadari kembali dan mengingat aturan pada point satu bahwa kamu seorang user baru dan gunakanlah jenis distro yang umum untuk pemula, nikmati tahapannya kemudian setelah paham baru bisa mencoba untuk bermain-main dengan distro Linux untuk tujuan spesifik misalkan kali linux untuk belajar penetration testing.

  3. Pahami konsep dasar sistem operasi Open Source mulai dari Pemaketan aplikasi dan aturan Direktori.

    Ini belum masuk ke perintah dasar Terminal pada Linux, tapi hal terpenting menurut saya pahami dahulu konsep pemaketan aplikasi mulai dari ekstensi file aplikasi linux menggunakan apa ? lalu untuk instalasi aplikasi online maupun offline harus paham, paham konsep dasar pengetahuan mengenai repository yang digunakan pada sistem operasi linux, kemudian paham masing-masing direktori pada sistem operasi linux dan fungsinya untuk apa.

    Tahapan ini sunggung butuh konsentrasi belajar, karena disini saya rasa sangatlah penting. Setelah paham pada point ke 3 maka untuk menguasai berbagai jenis distro linux akan lebih mudah.

  4. Kamu yang install Linux dual boot dengan sistem operasi Windows tapi bajakan, babat habis dan install saja single OS Linux saja!

    Dengan alasan belum bisa meninggalkan aplikasi-aplikasi di Windows, aplikasi di windows belum ada penggantinya jika migrasi ke linux. ini semua bagi saya pernyataan yang sedikit kurang tepat si, karena berbagai aplikasi alternatif yang ada di Windows kita bisa temui juga di sistem operasi Linux. Jika alasan dual boot Linux dan Windows untuk mempelajari Linux sembari memahaminya lalu menghapus windows setelah paham itu masih bisa di terima alasannya. Namun point pentingnya adalah kesadaran kamu menggunakan open source untuk tujuan meninggalkan sistem operasi bajakan dan menggunakan linux untuk ivestasi ilmu tambahan karena dengan menggunakan sistem operasi open source maka kamu akan menjadi lebih terampil.

  5. Temukan selah setelah terbiasa menggunaka Linux lalu cobalah bermain-main dan berkenalan dengan command line linux dengan aplikasi console Terminal.

    Kamu sudah terbiasa dan memahami point 1 sampai 4 diatas ? selamat kamu naik level. Saatnya kamu biasakan untuk mengenal perintah-perintah dasar Terminal di Linux dan pahami fungsi masing-masing perintah tersebut, lalu cobalah untuk di biasakan menggunakan perintah terminal di Linux untuk kebiasaan sehari-hari misalnya untuk hal kecil seperti membuat folder, menghapus folder/aplikasi, mengatur hak akses file. Pelan-pelan saja dan mulai biasakan pekerjaan-pekerjaan kecil itu diselesaikan dengan perintah di terminal.

    Ini akan membawa kamu untuk memperdalam kemampuan menguasai perintah-perintah pada terminal dimana pada akhirnya untuk urusan pekerjaan dengan kebutuhan yang lebih rumit dan merasa kompatibilitas dan efisiensi pekerjaan akan cepat dengan cukup mengeksekusi lewat perintah di terminal dan perlahan ini akan membawamu menuju kebiasaan di luar kepala terhadap perintah-perintah terminal di linux untuk kebutuhan komputasi sehari-hari.

Lima point diatas sebagai pengalaman saya merupakan modal dasar seorang pemula yang baru migrasi atau memutuskan untuk memilih linux sebagai sistem operasi pilihannya, kemudian saya akan melanjutkan atikel series Linux untuk pemula pada **Seasons II**
