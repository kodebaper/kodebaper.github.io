2. Untuk Pertama Kalinya Deploy Hugo di GitHub Page

Pertama kali melakukan Deploy blog static site Hugo saya ini ke repository GitHub yang sudah saya buat dan saya persiapkan sebelumnya, saya sempat mengalami beberapa error.

Adapun error yang saya alami selama proses Deploy telah saya rangkum, semoga beberapa [error-solved] yang saya dokumentasikan di blog ini dapat membantu untuk siapa saja yang sedang melakukan deploy kemudian menemui kendala dengan permasalahan yang sama seperti yang saya temui. Walaupun sebenarnya saya tidak secara lengkap mendokumentasikan error karena terlalu banyak error yang saya temui waktu itu sehingga saya tidak sempat mencatat satu persatu dengan rinci setiap error pada saat saya melakukan deploy ke GitHub page. Akan tetapi sejauh ini apa yang saya dokumentasikan ini akan cukup membantu untuk permasalahan seputar langkah melakukan deploy hugo static site ke GitHub page, dan berikut ini beberapa error yang saya temui selama proses Deploy Hugo ke github page.

3. Permasalahan pada repository yang telah ada

Ini adalah permasalahan pertama yang saya temui, dimana saya mengalami kendala saat mencoba melakukan deploy pertama kali ke github page, saya menemui masalah dengan pesan yang tertera seperti di bawah ini.

$ git remote add origin git@github.com:mazzempong/blog.git [perintah yang saya masukan]
  fatal: remote origin already exists [pesan errornya]

Setelah mencari beberapa saat untuk memperbaiki error tersebut, akhirnya saya menemukan untuk mengatasi permasalahan dengan pesan seperti diatas, saya menemukan solusi ini di situs stackoverflow berikut untuk mengatasinya.

$ git remote rm origin

Dan permasalahan pun selesai dengan baris perintah ini.

4. Gagal clone repository ke dalam direktori public

Ini adalah masalah yang cukup panjang penyelesaiinya yang membuat saya stuck pada step ini, dimana pada tahap ini saya harus mencari kesana kemari artikel yang memberikan solusi untuk permasalahan yang saya temui. Pada akhirnya saya menemukan solusinya setelah berselancar kesana kemari untuk hal ini.

pertama, pada tahap ini saya ingin melakukan clone dari repository yang saya buat di github dengan nama repositorynya yaitu kodebaper.github.io dimana repository ini sebelumnya adalah kosong, lalu saya buatkan submodule dari repository init.

Nah loh, repository intinya yang mana ?….

Repository intinya adalah, blogbackup [ini repository inti yang saya buat]

Jadi ada 2 repository yang saya siapkan yaitu repo blogbackup dan kodebaper.github.io, disini semua berawal dimana saat saya berada di direktori blog saya, saya mencoba untuk menambahkan submodule dimana submodule tersebut akan melakukan clone repository kodebaper.github.io ke dalam direktori public, namun apa yang terjadi pada saya ? banyak sekali saat itu saya dapati error yang tidak mau push ke direktori public.

Tahap ini saya lupa Dokumentasikan setiap pesan error
[saking banyaknya error]

Oke, tidak masalah karena pada akhirnya saya menemukan solusi dari sekian banyak artikel yang saya selami untuk mengatasi permasalahan tersebut, saya lagi lagi berterimakasih kepada stackoverflow karena ada salah satu artikel yang membantu sehingga blog ini dapat ter-deploy di GitHub page. Langkahnya adalah :

pastikan tetap di direktori blog kamu

Kemudian jalankan perintah Hugo, dimana ini akan menghasilkan sekumpulan file web statis di dalam direktori public

# hugo

Setelah itu kita masuk ke direktori public untuk melakukan push

# cd public/

Kemudian saatnya kita melakukan clone repository kodebaper.github.io ke dalam direktori public, dengan 3 baris perintah ini

# git add .
# git commit -m "pesan commit kamu"
# git push -u origin master

Setelah selesai, saatnya kita mengecek repository username.github.io (sesuaikan repo ini dengan username github kamu ya) seharusnya sejauh ini repo kamu sudah terisi dengan file blog static hasil clone tadi.

5. FYI untuk langkah ini

FYI ! langkah ini kita lakukan apabila dalam keadaan seperti yang saya alami dimana kita tidak bisa melanjutkan perintah untuk membuat submodule dengan perintah seperti di bawah ini.

git submodule add -b master git@github.com: [username-kamu]/[username-kamu].github.io.git public

Mungkin apabila kamu menemui hal yang sama, maka panduan ini bisa menjadi solusi, selamat mencoba !
