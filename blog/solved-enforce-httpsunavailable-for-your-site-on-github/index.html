Solved : Enforce HTTPS—Unavailable for Your Site

Dua hari setelah proses custom domain untuk blog saya ini di github page, setelah menunggu proses propagasi DNS yang memang cukup memakan waktu biasanya sampai 1x24 jam dan melakukan beberapa konfigurasi terkait dengan DNS di cloudflare, saya menemui masalah baru dimana blog saya ini untuk SSL dari sisi pengaturan di GitHub tidak bisa di aktifkan alhasil saya hanya bisa mengaktifkan SSL blog dari fitur yang ada di cloudflare namun tidak untuk di pengaturan GitHub Page, lalu dampaknya apa saja jika seperti ini ?

Adapun untuk dampak yang di dapatkan dari permasalahan seperti yang saya alami ini seperti, domain masih di kenal dengan HTTP belum HTTPS saat di buka menggunakan browser dimana di fitur pengatruan GitHub Page masih di sebutkan bahwa Your site is published at http://kodebaper.com/ bukan Your site is published at https://kodebaper.com/. Kemudian saat membuka blog saya ini menggunakan koneksi internet di kantor harus melalui verifikasi yang menyatakan laman yang di buka tidak secure, saya juga kurang paham dengan perangkat firewall Fortigate di kantor tempat saya bekerja melakukan verifikasi seperti ini terhadap blog saya.

Adapun untuk pemecaham permasalahan untuk mengatasi permasalahan ini sebagai berikut :

  1. Hapus terlebih dahulu domain di GitHub Page

    Saya menghapus terlebih dahulu nama domain yang sudah saya input di pengaturan GitHub Page hapus lalu kita simpan, biarkan saja kosong dahulu. Contoh pada gambar di bawah ini, saya menghapus domain kodebaper.com ---> lalu SAVE “sesuikan dengan nama domain kamu untuk di hapus terlebih dahulu ya”

    Sampai disini selesai untuk urusan pengaturan domain pada GitHub Page.

  2. Mengganti ulang Management DNS di CloudFlare

    setelah itu saya menuju pengaturan pada Management DNS di CloudFlare karena dari beberapa referensi yang saya baca ternyata tidak sama dengan pengaturan dari sisi DNS yang saya lakukan, kemudian saya melakukan perubahan DNS di CloudFlare menjadi seperti gambar di bawah ini.

    Gambar setting DNS CloudFlare yang benar

    Gambar yang di tampilakn saat ini merupakan setelah konfigurasi berhasil untuk merubah settingan HTTPS pada GitHub Page, untuk itu saya akan jelaskan dari ketiga pengaturan DNS sebelum ke pengaturan seperti gambar di atas.

    Settingan sewaktu masih salah
<Type>      <Name>          <Content>           <TTL>    <Proxy Status>
A           www             185.199.108.153      Auto    Proxied
CNAME       kodebaper.com   kodebaper.github.io  Auto    Proxied
TXT         kodebaper.com   google-site......    Auto    DNS Only

    Dari pengaturan untuk DNS pada CloudFlare seperti ini tentunya kurang tepat, dan mengakibatkan tidak bisanya fitur HTTPS di aktifkan pada pengaturan custom domain di github page untuk itu kembali saya pelajari dokumentasi yang banyak bertebaran di internet dalam pengaturan antara custom domain pada GitHub dan Management DNS di CloudFlare. Oke sampai disini selesai, untuk perubahan bisa melihat Gambar setting DNS CloudFlare yang benar seperti gambar di atas. Ini merupakan gambar yang berhasil mengaktifkan HTTPS pada GitHub Page.

  3. Kembali ke Pengaturan custom domain di GitHub Page

    Setelah pada poin dua di atas selesai kini yang saya lakukan adalah mengisi kembali nama domain yang sebelumnya tidak mau men’Ceklist’ pengaturan Enforce HTTPS, setelah di isi lalu simpan, setelah itu kita mesti menunggu untuk perubahan menjadi HTTPS dan fitur ini pun belim bisa di ceklist karena butuh waktu. Setelah menunggu sekitar 20 Menit saya refresh kembali page custom domain di GitHub dan fitur Enforce HTTPS saya sudah bisa di aktifkan dengan cara memberikan ceklis pada bagian Enforce HTTPS dan merubah keterangan pada custom domain GitHub Page menjadi Your site is published at https://kodebaper.com/.

Sampai pada ke tiga proses ini selesai, dan untuk permasalahan seperti ini tidak selalu di hadapi. Karena di beberapa kasus untuk custom domain di GitHub Page ada yang berjalan dengan lancar tanpa melewati proses seperti ini. Semoga bermanfaat dan semangat belajar!
