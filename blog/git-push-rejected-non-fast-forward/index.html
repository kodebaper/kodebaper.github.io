Git push rejected "non-fast-forward"

beberapa hari yang lalu saya sempat mengalami error setelah melakukan custom domain di github page, dimana ketika perubahan blog ini saya lakukan dan ingin melakukan update perubahan tersebut dengan cara push hasil perubahan ke repository github namun di tolak, kurang lebih untuk pesan error seperti di bawah ini :

Perintah git untuk untuk push perubahan sebagai berikut

# git push -u origin master

Pada step ini saya mendapati error kurang lebih seperti di bawah ini :

error: failed to push some refs to 'https://github.com/kodebaper/kodebaper.github.io.git'
hint: Updates were rejected because the remote contains work that you do
hint: not have locally. This is usually caused by another repository pushing
hint: to the same ref. You may want to first integrate the remote changes
hint: (e.g., 'git pull ...') before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

Sedikit penjelasan dari yang saya baca dari Stackoverflow untuk kasus ini, bahwa perubahan dari blog untuk di push ke repository github di tolak karena ada indikasinya repository lain yang terdorong ke repository sebelumnya, disini mungkin penyebabnya saya melakukan perubahan terhadap file .toml dimana saya merubah domain saya yang semula https://kodebaper.github.io menjadi https://kodebaper.com. Tapi tidak perlu khawatir karena disini ada penyelesaian singkatnya melalui perintah dibawah ini.

git fetch origin master:tmp
git rebase tmp
git push origin HEAD:master
git branch -D tmp

setelah ini di masukan saatnya kita tes dengan melakukan perubahan kembali terhadap blog, kemudian seperti biasa push ulang ke Github Page dengan cara seperti berikut :

hugo
cd /public
git add .
git commit -m "catatan push"
git push -u origin master

selanjutnya untuk tahapan ini semestinya sudah bisa berjalan normal, demikian artikel kali ini semoga bermanfaat !
