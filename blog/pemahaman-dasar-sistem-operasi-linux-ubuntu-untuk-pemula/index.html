Memahami Dasar Sistem Operasi Linux

Pemahaman dasar mengenai sistem operasi Linux Ubuntu telah saya praktekan sejak awal saya mulai mengenal sistem operasi Linux, hal ini akan sangat penting karena bagi saya sebelum memahami lebih jauh sistem operasi ubuntu maka ini suatu pondasi yang kuat untuk memahami Ubuntu dari hal yang paling dasar. Semua yang saya lakukan merupakan metode yang saya pakai dulu saat mulai berkenalan dengan sistem operasi ubuntu, jadi memang ini benar-benar saya lakukan dan cara seperti ini saya pelajari sendiri dalam pembelajaran awal-awal saya saat itu.

Saya mesti tau cara install aplikasi di Ubuntu

Bagi saya ini sudah menjadi hal dasar yang mesti kita ketahui terlebih dahulu, yaitu memahami bagaimana menginstall sebuah aplikasi di sistem operasi ubuntu kita.

  1. Menginstall aplikasi melalui Software Center, ini merupakan langkah pertama yang mesti kamu ketahu bahwa install Aplikasi dengan software center merupakan langkah termudah untuk instalasi sebuah aplikasi di ubuntu kamu. Tapi masalah baru pun muncul jika aplikasi yang hendak kamu install tidak tersedia di software Center.

  2. Menggunakan Terminal, Ini merupakan alternatif lain selain install aplikasi menggunakan Software Center dimana aplikasi untuk sistem operasi Ubuntu kamu bisa di install menggunakan terminal, ini juga berlaku untuk alternatif jika aplikasi yang hendak kamu install tidak tersedia di Softare Center.

  3. Menggunakan aplikasi yang di download dari internet, dengan ekstensi .deb, ini juga merupakan cara lain untuk install aplikasi di ubuntu selain menggunakan Software Center, dan Terminal. Dimana kamu dapat menginstall aplikasi yang hendak kamu pasang di ubuntu dengan menggunakan alternatif ini, dan ini merupakan cara yang paling saya suka karena hampir untuk keseluruhan aplikasi yang kita butuhkan untuk di install di ubuntu kita tersedia di internet dengan ekstensi .deb tentunya karena kita menggunakan sistem opersi Linux ubuntu. Namun kita juga mesti memperhatikan aplikasi berekstensi .deb yang kita download sesuai dengan sistem operasi ubuntu kita ya, menggunakan arsitektur 32 bit atau menggunakan 64 bit.

Saya mesti tau perbedaan user biasa atau user akses penuh/root user

Ini berlaku jika kamu sudah mulai mengenal aplikasi command shell dengan menggunakan Terminal di ubuntu kamu, jadi untuk default begitu kamu jalankan aplikasi Terminal di ubuntu kamu biasanya kamu akan menempati hak akses dengan user bias, itu berarti kamu tidak memiliki hak akses penuh untuk beberapa direktori atau file kamu tidak bisa melakukan perubahan apaun, namun sebaliknya dengan kamu menjadi root user maka kamu akan memiliki hak akses penuh sehingga kamu dapat melakukan perubahan atau mengeksekusi suatu file di dalam sisitem operasi ubuntu kamu. Untuk dapat menjadi root user kamu cukup mengetikan sudo su lalu masukan password sistem operasi kamu [untuk lebih jelasnya kamu dapat membaca artikel yang saya tulis di sesion 6 kelanjutan dari artikel sesion ini]

Saya mesti tau apa itu repository di ubuntu

Jadi di sistem operasi ubuntu ada yang namanya Repository, dimana repository ini berfungsi sebagai tempat penyimpanan sistem, dimana nantinya Kamu mengambil dan menginstal pembaruan serta aplikasi. Nah untuk setiap repositori merupakan kumpulan perangkat lunak yang di-host di server jauh dan dimaksudkan untuk digunakan untuk menginstal dan memperbarui paket perangkat lunak pada sistem Linux ubuntu kamu. kamu juga bisa mengeceknya repository kamu atau bahkan mengganti repository dengan mirror terdekat yang kamu tuju, namu untuk mengubah repository kamu perlu memahami betul apa itu repository serta fungsinya itu sendiri. Untuk melihat repository pada sistem operasi Linux ubuntu kamu bisa melalu aplikasi terminal lalu ketikan perintah # nano /etc/apt/source.list

Tiga hal ini merupakan modal dasar saya awal mengenali dan belajar Ubuntu, dan untuk menunjang pemahaman lain tentunya harus banyak membaca artikel di internet yang di rekomendasikan untuk pemula linux dan jangan lupa untuk langsung mempraktekannya. Selamat membaca !
