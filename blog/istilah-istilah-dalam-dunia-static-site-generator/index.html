Istilah yang sering di gunakan dalam dunia static site generator

1. Prakata dalam pembahasan Artikel ini

Pada pembahasan kali ini saya akan melakukan pembahasan dengan istilah-istilah yang sering digunakan dalam dunia static site generator, lebih tepatnya istilah ini yang sering digunakan untuk static site generator yang saya gunakan yaitu Hugo. Sebenarnya untuk di sebut istilah yang sering di gunakan dalam dunia static site generator saja juga tidak, karena istilah-istilah ini juga sering di gunakan untuk dunia website baik yang menggunakan engine CMS ataupun jenis website Dinamis.

Isi dari seluruh penjelasan pada pembahasan artikel kali ini tidak hanya murni dari hasil pengetahuan saya dan argunemtasi saya semata, namun kebanyakan dari penjelasan dari artikel ini saya dapatkan dari beberapa sumber yang menurut saya sangat mudah untuk di pahami dan membuat saya paham untuk istilah-istilah dalam satic site generator yang saya gunakan.

Lalu apa yang pembaca dapatkan dari artikel ini ?

Ada baiknya kamu yang ingin membaca ini, untuk menengok ke samping pada daftar isi artikel untuk mengetahui apa saja isi dari pembahasan artikel yang akan kamu baca.

2. Pengertian static site generator

Secara umum saya tidak menemukan sejarah awal lahirnya teknologi static site generator, atau mungkin saya kurang mencari referensi untuk mengetahui hal ini. Disini saya hanya akan menjelaskan sedikit tentang apa itu static site generator

ini adalah penjelasan mengenai apa itu static site generator Hugo yang saya dapatkan dari laman resmi Hugo, atau lebih tepatnya pada situr resmi hugo DISINI 

PENGERTIAN HUGO DARI LAMAN RESMI
-----------------------------------------------------------------------------
Hugo is a fast and modern *static site generator* written in Go
and designed to make website creation fun again.

Hugo is a general-purpose website framework. Technically speaking,
Hugo is a *static site generator*. Unlike systems that dynamically 
build a page with each visitor request, 
Hugo builds pages when you create or update your content. 
Since websites are viewed far more often than they are edited, 

Hugo is designed to provide an optimal viewing experience for your website’s
end users and an ideal writing experience for website authors.

Kalau bagi saya sendiri memahami pengertian static site generator dari pengertian diatas yaitu:


Sebuah alat/teknologi yang dapat meng-generate source code untuk dapat menghasilkan static website dimana ini artinya teknologi static website tidak membutuhkan pengaturan dan konfigurasi dari sisi BackEnd atau konfigurasi pengaturan pada sisi server secara spesifik, itu artinya static website langsung memberikan html ke setiap pengunjung atau yang mengaksesnya.

3. Peruntukan penggunaan static site

Lalu untuk siapa static site generator ini di buat, atau di peruntukan ?

Lebih tepatnya static site generator menurut saya cocok di gunakan untuk personal yang suka menulis di blog dengan design dan fitur yang simple, di gunakan untuk membuat website landing page serta blog untuk media portfolio kita.

Namun apabila ingin memiliki website yang kaya akan fitur dan lebih kompleks, penggunaan static site generator tidak di anjurkan. Ini juga salah satu alasan saya menggunakan static site dari Hugo karena saya memang pure menggunakan blog ini untuk membuat catatan-catatan dari hasil pelajaran yang sudah saya pelajari. Selebihnya hanya sebagai media menyalurkan hobi karena saya juga senang menulis.

4. Kekurangan dan kelebihan static site

Adapun untuk kelebihan dan kekurangan dari static site generator dari hugo sebagai berikut:

Kelebihan *static site generator*

  • Proses load halaman yang jauh lebih cepat Bagaimana tidak, hugo hanya menggunakan halaman statis, dimana ini akan mempercepat proses loading dibandingkan dengan halaman dinamis yang memerlukan pembacaan script dan database.

  • Murah Biaya Untuk dapat mempublish halaman statis tidak memerlukan hosting yang harga mahal, karena tidak memerlukan hosting yang dilengkapi dengan database. Bahkan banyak yang menyediakan hosting gratis untuk SSG ini, saya sendiri memilih layanan gratis dari GitHub Page untuk mempublish halaman statis blog ini. Lalu saya hanya mengeluarkan biaya untuk menambahkan custom domain ke layanan GitHub Page nya. Murah bukan ?

  • Keamanan Terjaga Halaman statis keamanannya lebih terjaga, walaupun ini juga bukan sebuah jaminan, namun perlu di ketahui bahwa static site ini tidak berhubungan dengan kode dan database yang dapat ditembus celah keamanannya.

Kekurangan *static site generator*

  • Tidak Dinamis Yang ini sudah jelas dimana pemilik website tidak dapat dengan cepat dan mudah untuk melakukan update data, karena tidak dilengkapi dengan admin panel atau back end. itu yang salah satu penyebabnya seorang bloger dengan website statis saat mengupdate konten harus menulis dan membuild ulang halamannya setiap kali ada perubahan terhadap blog tersebut.

  • Memiliki Skils Untuk memulai pembuatan static site generator, memerlukan Setup awal yang tampilannya berupa command melalui terminal. Namun ini juga bukan suatu syarat mutlak bahwa harus memiliki skill dasar yang mumpuni untuk hanya sekedar bisa membuat website/blog dengan static site generator, karena awam sekalipun bisa melakukannya dimana dokumentasi yang membahas mengenai static site generator saat ini sudah sangatlah banyak di internet.

5. Jenis-jenis Platform static site generator

Pada dasarnya saat ini banyak sekali pilihan static site generator yang dapat menjadi pilihan untuk siapa saja yang ingin memiliki blog/website menggunakan teknologi static site generator. Adapun static site generator yang dapat di pilih antara lain:

Hugo, Next.js, Gatsby, Jekyll, Hexo, Slate, dll. Semua memiliki kelebihan dan kekurangan masing-masing. nah sekarang framework mana yang akan kamu gunakan ?
dimana kebanyakan orang akan memilih framework berdasarkan bahasa pemrograman yang di gunakan sesuai dengan kemampuan juga setiap orang sudah memahami bahasa pemrograman apa. Hal ini penting karena akan memiliki dampak kedepan terhadap perkembangan blog/website yang di kelolanya.

Mengapa memilih Hugo ?

Disini saya memilih static site generator dari Hugo bukan karena saya memiliki pemahaman tentang bahasa pemrograman Golang, ya hugo ini di bangun dengan bahasa yang di ciptakan oleh Google yaitu bahasa golang. Alasan utama saya memilih hugo karena saya tertarik dengan golang, dari beberapa yang saya baca juga golang memiliki hasil yang sangat bagus di beberapa benchmark di banding dengan bahasa pemrograman lain.

6. Istilah-istilah dalam dunia static site generator

Berikut merupakan istilah-istilah yang sering di gunakan dalam dunia static site antara lain sebagai berikut :

  • Mark Down

Markdown merupakan sebuah tools untuk melakukan konversi text-to-HTML yang bisa digunakan ketika membuat konten web, nantinya kita akan sering berurusan dengan yang namnya Markdown sebagai pengguna static site, salah satu contohnya yaitu penulisan artikel pada Hugo akan menggunakan file berekstensi .md atau Markdown

  • file config.toml

Ini bukan sebuah istilah namun sengaja saya sematkan dimana nama file config.toml tidak akan asing lagi untuk pengguna static site generator dari hugo, namun ada opsi tidak harus file dengan ekstensi .toml saja karena hugo juga menyediakan eksetensi untuk file konfigurasinya dengan eksetensi .YAML ATAUPUN .JSON. File config ini merupakan file konfigurasi default pada root hugo untuk dimana biasanya digunakan untuk mengatur konfigurasi penamaan domain, theme, comment atau description pada blog hugo dan masih banyak lagi.

  • Homepage/index.html, Halaman Artikel/single.html dan Halaman Taxonomy/list.html

sekalipun ini bukan sebuah istilah namun pada umumnya pengguna static site Hugo harus memahami 3 halaman ini dalam sebuah static website.

  • Front Matter

Front matter memungkinkan Anda untuk menyimpan metadata yang dilampirkan ke sebuah instance dari jenis konten yang disematkan di dalam file konten, front matter adalah data-data yang tersimpan untuk konten kita dimana biasanya berisi informasi meta dari file tersebut. Isinya sebagai contoh untuk memberi judul, tanggal dan opsi pilihan apakah postingannya ingin dalam draft atau dipublish.

  • Slug

Slug merupakan sebuah teks yang akan diletakkan ataupun muncul setelah nama domain dan menjadi bagian dari permalink dimana ini akan berfungsi untuk mengarahkan pengunjung website masuk ke konten kamu, untuk mengetahui lebih jauh apa itu slug kamu tidak perlu khawatir karena sudah di dokumentasikan dan di jelaskan pada halaman dokumentasi resmi website Hugo.
