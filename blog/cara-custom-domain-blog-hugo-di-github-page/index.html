Cara Custom Domain Blog Hugo Di GitHub Page

Beberapa waktu yang lalu saya telah menuliskan artike tentang 3 persiapan sebelum melakukan custom domain untuk blog hugo di layanan GitHub page, nah untuk kesempatan kali ini saya akan membahas mengenai bagaimana caranya melakukan konfigurasi custom domain di GitHub page. untuk itu artikel saya buat urutan daftar langkah pengerjaanya. Untuk di kasus ini saya anggap kamu sudah melakukan dua hal ini sebelum kamu mulai untuk mempraktekannya.

Pertama yaitu sudah memiliki domain aktif yang akan di masukan di konfigurasi ke Github Page

Kedua kamu sudah mendaftar dan memiliki Account aktif di Cloudflare

Pembahasan saya mulai dari urutan Langkah 1 s.d Langkah 4 dan berikut ini untuk langkah-langkahnya:

Langkah I pada sisi CloudFlare

Hal pertama yang harus kamu lakukan setelah domain yang di beli aktif, langsung login ke CloudFlare untuk mulai setup, kita mesti add site dari domain yang kita akan di terapkan pada CloudFlare. Seperti contoh gambar di bawah ini adalah domain yang sudah saya tambahkan

Untuk kamu yang belum menambahkan nama domain kamu, caranya sangat mudah kamu tinggal klik Add a site kemudian ikuti petunjuk sesuai dengan yang di berikan. Sampai pada proses yang menunjukan bahwa adanya Name Server yang di rekomendasikan dari CloudFlare bahwa Name Server pada penyedia domain harus di ganti dengan Name Server dari CloudFlare seperti gambar di bawah ini, oh ia disini juga petunjuk pemasangan Name Server dari Cloudflare ke pengaturan di layanan domain yang kita gunakan sudah di jelaskan dengan sangat lengkap. Untuk memudahkannya bisa lihat gambar di bawah ini.

Karena saya menggunakan layanan dari Domainesia maka saya harus mengganti Name Server dari domainesia seperti yang tertera pada petunjuk nomor 1 pada gambar, saya merubah ke Name server dari CloudFlare.

Langkah II pada sisi Penyedia Domain

Pada langkah ini yang akan kita lakukan hanya sebatas mengganti Name Server dimana sebelumnya untuk Name Server masih menggunakan bawaan default dari penyedia layanan domain, namun untuk kali ini kita akan mengganti NS (Name server) dengan yang sudah di rekomendasikan oleh cloudflare. Gambar tertera seperti pada langkah I dimana menunjukan instruksi untuk merubah DNS yang sesuai dengan petunjuk dari cloudflare.

Disini saya merubah Name Server di pengaturan dashboard domain dari Domainesia. Saya telah mengganti dan memasukan Name Server dari CloudFlare tersebut seperti yang tertera pada gambar diatas.

Langkah III pada sisi GitHub Page

Pada langka ke Tiga ini saatnya domain kita insert ke dalam konfigurasi dari GitHub Page nya, langkahnya pun mudah.

  1. Login ke GitHub
  2. Masuk ke Repository utama
  3. Kemudian ke menu Setting
  4. Scroll ke bawah sampai pada pengaturan GitHub Page, seperti gambar di bawah ini.
  5. isi saja pada form Custom Domain dengan nama domain Kamu
  6. Lalu klik Save

Pada proses ketiga ini, konfigurasi untuk custom domain di GitHub Page hampir selesai dengan menyisakan satu langkah lagi.

Langkah IV DNS Management pada CloudFlare

Adapun untuk langkah terakhir ini kita masuk kembali ke pengaturan pada cloudflare, caranya yaitu:

  1. Pastikan kamu sudah login ke CloudFlare
  2. pada tampilan awal Cloudflare Klik nama Domain kamu yang sudah di tambahkan seperti di jelaskan pada langkah satu di atas
  3. Kemudian pilih ke pengaturan DNS lalu sesuaikan dengan nama domain kamu ya, untuk memudahkan pengaturan DNS pada domain kamu, disini saya sertakan gambar seperti tertera di bawah ini dimana Management DNS pada CloudFlare pada gambar tersebut merupakan pengaturan untuk DNS domain saya. Kamu tinggal menyamakan sesuai gambar di bawah ini.

Dan untuk sampai proses ke 4 ini sudah cukup, namun ada satu lagi kamu mesti merubah nama domain juga pada file config.toml pada file Hugo kamu pada baris pertama baseURL

Ini adalah pengaturan base url pada file **config.toml** hugo saya.
*
Sebelum di ganti
----------------
baseURL = "https://kodebaper.github.io/"
*
Sesudah di ganti
---------------- 
baseURL = "https://kodebaper.com/"
