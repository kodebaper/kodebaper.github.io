Memanfaatkan Github Page Untuk Deploy Hugo

Beberapa waktu yang lalu saya sudah memposting artikel tentang perkenalan singkat blog pribadi saya ini yang menggunakan static site generator dari Hugo, nah untuk kesempatan kali ini saya akan mengenalkan alternatif yang saya gunakan untuk menghosting blog hugo saya secara gratis, setelah baca-baca dokumentasi yang tersedia di website resmi Hugo, ternyata saya menemukan cara yang sudah di dokumentasikan dengan sangat jelas oleh tim Hugo tentang bagaimana melakukan deploy hugo di layanan GitHub Page. Pada dasarnya banyak cara yang bisa kita lakukan untuk mendeploy Hugo ini ke github namun perlu menjadi catatan disini terkadang tidak semua cara yang terdokumentasi di website resmi hugo dapat kita ikuti.

Mengapa demikian ? ya, karena sepengalaman saya pada langkah tertentu terkadang kita akan menemui error contohnya saya sendiri. Ini bisa terjadi karena ada beberapa faktor dan lain hal, untuk itu dokumentasi yang saya tulis ini adalah dokumentasi dimana saya melakukan deploy Hugo ke GitHub setelah mengalami beberapa error.

Berikut merupakan langkah-langkah mendeploy Hugo ke GitHub page:

  • Pastikan kamu sudah membuat account di github melalui situs resminya https://github.com/

  • Kemudian sebelum melangkah lebih jauh ada baiknya seting SSH terlebih dahulu ke github agar saat push file ke repositori di github tidak repot. [optional ya, ini hanya saran saya]

  • Kemudian buatlah 2 Repositori di github, yaitu repositori:

    blogku [terserah mau di kasih nama apa]

    username.github.io

    Kurang lebih seperti gamabr di atas dengan nama repositori yang sudah kamu siapkan, lakukan 2 kali namun untuk yang keduanya, wajib ya buat nama Repositori username.github.io sesuaikan saja dengan username account GitHub kamu.

    Mungkin ada yang bertanya kedua repositori itu fungsinya untuk apa, oke disini akan saya jelaskan berdasarkan sepengetahuan saya. Jadi repositori yang pertama sebenarnya repositori optional yang akan kita maksudkan sebagai backup dari isi file hugo kita, kemudian untuk repositori kedua akan di pusatkan sebagai repositori hasil render / menyimpan file yang ada di public nantinya. Kurang lebih seperti ini pemahaman saya ya, mari kita lanjut ke langkah selanjutnya.

  • Pastikan kita sudah masuk ke direktori hugo kita, saya menggunakan terminal pada kasus ini dimana saya simpan file Hugo saya di direktori /Hugo Project

    admin$ [ini direktori pertama saya buka Terminal]
dimas admin$ [saya sudah pindah ke direktori Hugo saya]

  • lakukan inisialisasi git terlebih dahulu dengan mengetikan perintah

    git init

  • Kemudian kita tambahkan repositori remote nya ,pada kasus ini saya menggunakan SSH

    git remote add origin git@github.com:UserNameKamu/NamaRepoKamu.git

  • sampai tahap ini seharusnya belum ada kendala, repositori lokal sudah selesai di persiapkan sekarang saatnya kita upload ke GitHub dengan 3 perintah ini

    /git add .
/git commit -m "kasih pesan commit kamu disini ya"
/git push -u origin master

  • Pada tahap ini seharusnya repositori yang pertama kali di buat tadi di awal sudah terisi file hugo kita.

  • Kemudian pada langkah ini saya menemukan cara yang sedikit berbeda dimana seharusnya kita membuat submodule dari repositori intinya yaitu [repositori blogku], mengapa mesti seperti ini ? karena, repositori [usernamekamu.github.io] masih kosong belum ada isinya itu sebabnya saat kita akses [https://usernamekamu.github.io] masih belum bisa di akses blog Hugo kamu. Cara ini saya temukan dari artikel di stackoverflow karena pada langkah ini dimana saya memasukan perintah

    git submodule add -b master git@github.com:{username}/{username}.github.io.git public

Terjadi error tidak mau membuat submodule [lupa saya tidak sempat catat pesan errornya waktu itu] untuk itu saya hanya menggunakan 5 langkah ini

    *Pertama masukan perintah*
    \hugo
    *Kemudian masuk direktori /public*
    cd /public
    *Kemudian kita commit dengan perintah*
    \git commit -m "isi pesan commit kamu"
    *Kemudian saatnya kita push*
    \git push -u origin master

Sekarang coba cek repositori di github kamu, seharusnya repositori [usernamekamu.github.io] sudah terisi file hugonya, cara ini terbukti berhasil karena saya sudah menggunakan cara ini sampai blog hugo saya saat ini dapat di akses. Satu lagi jangan sampai lupa sebelumnya tadi kita mesti mengganti URL di file config.toml dengan url repo di github kita.

isinya kurang lebih seperti ini.

baseURL = "https://usernameKamu.github.io/" [sesuaikan dengan punya kamu]
languageCode = "en-US"
title = "blog ku"
theme = "namathemekamu"

Sebelum artikel ini saya akhiri, perlu di ingat bahwa setiap kali kita melakukan perubahan blog lalu kita akan mengupdate untuk bisa di akses di internet kita mesti selalu memasukan perintah ini.

/git add .
/git commit -m "kasih pesan commit kamu disini ya"
/git push -u origin master
